# zenoh-flat-c

A C binding for [`zenoh-flat`](../zenoh-flat), generated by
[`prebindgen`](../prebindgen)'s `lang::Cbindgen` adapter. It exposes a **dual C
API**:

1. **Flat API** — `include/zenoh_flat.h`. The generated `z_flat_*` ABI: bare
   opaque-pointer handles, typed `_drop`s, by-value closure structs. This is the
   direct, low-level surface.
2. **zenoh-c-compatible API** — `include/zenoh.h` (+ `zenoh_commons.h`,
   `zenoh_macros.h`, `zenoh_constants.h`, `zenoh_memory.h`). A **header-only**
   layer that re-exposes the [zenoh-c](https://github.com/eclipse-zenoh/zenoh-c)
   API (`z_owned/loaned/moved/view`, the `_Generic` `z_loan`/`z_drop`/`z_move`/
   `z_closure` macros, options structs, `z_result_t`) on top of the flat ABI, so
   existing zenoh-c examples/tests compile and run unmodified.

Both share one library, `libzenoh_flat_c.{a,dylib,so}`. The compat layer is all
`static inline`, so it compiles into the consumer.

## Layout

```
include/
  zenoh_flat.h       generated flat API (z_flat_*)            [gitignored]
  zenoh.h            zenoh-c-compatible umbrella (include this)
  zenoh_commons.h    owned/loaned/moved/view types, inline wrappers, closure bridge
  zenoh_macros.h     z_loan / z_loan_mut / z_move / z_drop / z_closure  (_Generic)
  zenoh_constants.h  z_result_t codes, enum value names, config keys
  zenoh_memory.h     z_malloc / z_free / z_sleep_*
src/lib.rs           include!()s the prebindgen-generated wrappers
build.rs             runs prebindgen + cbindgen; writes include/zenoh_flat.h
examples/            zenoh-c closure-core examples (ported verbatim)
tests/               supported subset of zenoh-c tests (ported verbatim)
CMakeLists.txt       builds the lib (via cargo) + examples + tests (ctest)
```

`include/zenoh_flat.h` is produced by `build.rs` on every `cargo build` and is
gitignored; the compat headers are committed.

## Building

Plain cargo (library only):
```
cargo build
```

CMake (library + examples + tests, modeled on zenoh-c):
```
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build --output-on-failure
```
The CMake build runs `cargo build` through a custom target, then links the
examples/tests against the `zenohflatc::lib` IMPORTED target (dual-API include
dir). `BUILD_SHARED_LIBS=ON` (default) links the dylib; `OFF` links the static
lib (which pulls in the Rust/zenoh native deps).

## How the compat layer bridges to the flat ABI

| zenoh-c | zenoh-flat-c | bridge |
|---|---|---|
| `z_owned_X_t` (stack) | `z_flat_X_t*` (heap handle) | `z_owned_X_t { z_flat_X_t* _0; }` |
| `z_loaned_X_t*` | same handle | `typedef z_flat_X_t z_loaned_X_t;` |
| `z_moved_X_t*` | by-value consume | `{ z_owned_X_t _this; }` + cast |
| one closure `{call,drop}`, **loaned** arg | two closures `(callback, on_close)`, **owned** arg | heap `_z_bridge_t`: per-msg trampoline casts owned→loaned then drops it; on_close runs the zenoh-c drop once + frees the bridge |
| `z_result_t` | `z_flat_error_t{char*}` + NULL/bool | any flat error → `Z_EGENERIC`, message freed |
| `..._options_t*` (nullable) | positional args | translated; zenoh defaults applied on NULL |
| borrowed loan accessors | `const z_flat_X_t*` (borrow) | direct, no per-message alloc |

## Ported examples (all run)

Closure-core: `z_put`, `z_delete`, `z_pub`, `z_sub`, `z_queryable`, `z_scout`,
`z_info` — copied unmodified from zenoh-c.

## Ported tests (ctest, all pass)

- `z_api_keyexpr_test` — keyexpr equals/includes/intersects/canonize/declare.
- `z_api_session_test` — open / drop / close.
- `z_leak_pub_sub_test` — two-session in-process pub/sub (functional + leak).

## Not yet supported (skipped tests/examples)

The compat layer covers the closure-based core. These zenoh-c features are not
implemented, and tests/examples depending on them were not ported:

- **`z_get` + channels** (`z_fifo_channel_*`, `z_recv`) and `z_querier`.
- **Timestamps + `source_info`** (`z_int_pub_sub_test`).
- **Bytes serialization / reader / writer / iterators**, `z_bytes_clone`.
- **Encoding** `from_str` / `equals` / `set_schema`.
- **Liveliness**, **shared memory (SHM)**, **advanced/`ze_*`**, **cancellation
  tokens**, transport/link introspection.

### Known limitations (within the supported subset)

- `z_keyexpr_as_view_string` and `z_query_parameters` materialize an *owned*
  string into a `z_view_string_t`, which zenoh-c never drops → a small leak per
  call. `z_view_keyexpr_from_str` allocates a flat keyexpr the view never frees.
  The handle hot path (`z_sample_payload`, etc.) is leak-free (borrowed
  accessors). Fix = a borrowed `&str` (ptr,len) output in prebindgen.
- Matching-status listeners are a no-op stub. `z_close` is a no-op (the real
  close happens on `z_drop`). `Z_FEATURE_UNSTABLE_API` is left undefined.
